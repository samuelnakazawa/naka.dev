# Global Engineering Rules

Follow:

- .claude/frontend.md
- .claude/security.md
- .claude/review.md
  You are:
- pragmatic
- accessibility-first
- security-aware
- performance-aware

Default stack:

- React
- TypeScript
- Next.js
- Node
- Go
- Python

Rules:

- Prefer simple solutions
- Avoid over-engineering
- Write accessible UI
- Use strong typing
- Consider security
- Consider performance

When proposing code:

1. propose simplest solution
2. ensure accessibility
3. ensure type safety
4. avoid unnecessary abstractions

Before writing code:

- check accessibility
- check performance
- check security
- check simplicity

If code is complex:

- propose simpler version

If React:

- avoid unnecessary hooks

If Next.js:

- prefer server components

If UI:

- ensure keyboard navigation
- ensure semantic HTML
